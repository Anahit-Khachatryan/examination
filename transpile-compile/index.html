Angular-ն օգտագործում է TypeScript, որը հանդիսանում է JavaScript-ի սուպերսեթ։ TypeScript-ն ունի հավելյալ հատկություններ, ինչպիսիք են ստատիկ տիպավորումը, դասերի հայտարարումը, և այլն։ TypeScript ֆայլերը (.ts) պետք է թարգմանվեն (transpile) JavaScript-ի ֆայլերի (.js)՝ որպեսզի դրանք կարողանան աշխատել զննարկիչներում: TypeScript Compiler (tsc) TypeScript Compiler-ը (tsc) հանդիսանում է հիմնական գործիքը TypeScript-ի ֆայլերը JavaScript-ի ֆայլերի թարգմանելու համար։ Այն ավտոմատ կերպով կարդում է tsconfig.json ֆայլը, որը պարունակում է TypeScript-ի կազմավորումների պարամետրերը: Compile Tools Angular-ում Angular-ում կան երկու հիմնական կոմպիլյացիայի մեթոդներ՝ JIT (Just-In-Time) և AOT (Ahead-of-Time): Just-In-Time (JIT) Compilation JIT կոմպիլյացիան տեղի է ունենում զննարկչում ծրագրի բեռնման պահին: Սա նշանակում է, որ ամբողջ TypeScript և Angular-ի կոդը բեռնվում և կոմպիլացվում է զննարկչում, երբ օգտատերը այցելում է ձեր վեբ կայքը: Առավելությունները՝ Արագ զարգացման ցիկլ. Ձեզ հարկավոր չէ կազմավորել (build) ծրագիրը ամեն փոփոխությունից հետո: Հեշտ է օգտագործել զարգացման ժամանակ, քանի որ թույլ է տալիս արագ տեսնել փոփոխությունների արդյունքները։ Թերությունները՝ Ավելի մեծ կոդի ծավալ բեռնման համար, քանի որ կոմպիլյատորը պետք է ներառված լինի: Ավելի երկար բեռնման ժամանակ, քանի որ կոմպիլյացիան տեղի է ունենում զննարկչում: Ahead-of-Time (AOT) Compilation AOT կոմպիլյացիան տեղի է ունենում ծրագրի կազմավորման (build) ժամանակ, ոչ թե զննարկչում: Սա նշանակում է, որ TypeScript և Angular-ի կոդը կոմպիլացվում է նախքան բեռնումը, և միայն կոմպիլացված JavaScript ֆայլերն են բեռնվում browser-ում: Առավելությունները՝ Ավելի փոքր կոդի ծավալ բեռնման համար, քանի որ կոմպիլյատորը ներառված չէ: Ավելի արագ բեռնման ժամանակ, քանի որ կոմպիլյացիան արդեն ավարտված է: Առավել անվտանգ կոդ, քանի որ սխալները հայտնաբերվում են կազմավորման ժամանակ: Թերությունները՝ Զարգացման ընթացքում ամեն փոփոխությունից հետո պահանջվում է վերակազմավորում: Ավելի բարդ է կարգավորել, քան JIT կոմպիլյացիան: npm start Երբ
