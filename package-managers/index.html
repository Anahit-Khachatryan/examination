Angular նախագծերում փաթեթների կառավարման համար հիմնականում օգտագործվում են երկու հիմնական package manager՝ npm (Node Package Manager) և yarn. Այս գործիքները թույլ են տալիս կառավարել dependencies (բացի այդ, նաև պլագիններ, լայբրարներ և այլ բաներ), որոնք անհրաժեշտ են Angular նախագծերի զարգացման համար: 1. npm (Node Package Manager) 2. Yarn 3. Pnpm Dependencies - Այս հատվածը պարունակում է այն փաթեթները, որոնք անհրաժեշտ են ձեր ծրագրի գործնական աշխատանքին, արտադրական միջավայրում: Սրանք այն լայբրարներն ու փաթեթներն են, որոնք ծրագիրը պետք է գործարկի ճիշտ: DevDependencies -Այս հատվածը պարունակում է այն փաթեթները, որոնք անհրաժեշտ են միայն զարգացման ընթացքում, բայց չեն պահանջվում արտադրական միջավայրում: Այսպիսով, սա սովորաբար ներառում է ինստրուկցիաներ, լիկեր, մոդուլներ, որոնք օգտագործվում են կոդի որակի ստուգման, կառուցման կամ մշակման համար: package-lock.json - Այս ֆայլերը ապահովում են, որ բոլոր աշխատակիցները և միջավայրերը օգտագործեն նույն կախվածությունների կոնկրետ տարբերակները, ինչը կանխում է տարբեր խնդիրներ տարբերակների միջև և ապահովում է կայունություն: Package managers-ը կարևոր են, որովհետև նրանք ավտոմատացնում են կախվածությունների տեղադրման, թարմացման և կառավարման գործընթացը, ինչը հեշտացնում է ծրագրի զարգացումը, զերծ է թողնում ձեռքով կախվածությունների կառավարման պարտականություններից, նվազեցնում է տարբերակների հակասությունները և օգնում է պահպանել կայունություն: --save: Այս դրոշը ավելացնում է փաթեթը dependencies-ի բաժին, ինչը նշանակում է, որ այն անհրաժեշտ է արտադրական միջավայրում: --save-dev: Այս դրոշը ավելացնում է փաթեթը devDependencies-ի բաժին, որը նշանակում է, որ այն անհրաժեշտ է միայն զարգացման ընթացքում:
